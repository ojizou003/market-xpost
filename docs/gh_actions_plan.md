現在、windowsのタスクスケジューラを使用して月～金のお昼12時（日本時間）にスクリプトを実行していますが、GitHub Actionsを使用して同様のスケジュールされたタスクを設定したいと考えています。以下は、そのための計画です。   
# GitHub Actionsでのスケジュールされたタスクの設定計画
## 目的
ローカルPCに依存せず、安定した環境でスクリプトを定期的に実行したい。
## スケジュール
月曜日から金曜日の毎日、12:00 PM（日本時間）にスクリプトを実行。
## 検討課題
- seleniumを使用しているためブラウザとドライバの相性の問題が発生する可能性があるので、dockerコンテナ内を使用した方がいいのではないか。
